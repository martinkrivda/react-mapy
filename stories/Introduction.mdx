# `react-mapy`

React map primitives and presets built on Leaflet, with Mapy.com as the default tile provider and a small, explicit public API.

## Install

```bash
pnpm add react-mapy react react-dom leaflet
```

Import Leaflet's CSS once in the host application:

```ts
import 'leaflet/dist/leaflet.css';
```

For clustered event markers, add the optional plugin:

```bash
pnpm add leaflet.markercluster
```

And load it once in the host app together with its CSS:

```ts
import 'leaflet/dist/leaflet.css';
import 'leaflet.markercluster/dist/MarkerCluster.css';
import 'leaflet.markercluster/dist/MarkerCluster.Default.css';
import { loadLeafletMarkerCluster } from 'react-mapy';

await loadLeafletMarkerCluster();
```

## Quick start

```tsx
import { LeafletMap, MapTileLayer } from 'react-mapy';
import { createMapyProvider } from 'react-mapy/providers/mapy';

const provider = createMapyProvider({
  apiKey: import.meta.env.VITE_MAPY_API_KEY,
  variant: 'outdoor',
});

export function Example() {
  return (
    <LeafletMap center={{ lat: 50.0755, lng: 14.4378 }} zoom={13} style={{ height: 420 }}>
      <MapTileLayer provider={provider} />
    </LeafletMap>
  );
}
```

## Direct and proxied Mapy tiles

`createMapyProvider(...)` supports both:

- direct frontend tile requests using `apiKey`
- backend-proxied tile requests using `proxy.urlTemplate`

Use the proxy mode in production if you do not want to expose your Mapy.com API key in browser code.

Storybook includes a dedicated example in `Leaflet/MapTileLayer -> MapyBackendProxyExample`.

For local live preview without exposing the key to the browser, point Storybook at your backend service:

```bash
STORYBOOK_MAPY_PROXY_TARGET=http://localhost:3001 pnpm storybook
```

The recommended endpoint shape is:

```text
/rest/v1/map/tiles/{variant}/{tileSize}/{z}/{x}/{y}?lang={lang}
```

By default, Storybook will proxy `/rest/v1/map/*` to that backend target. You can override the prefix with `STORYBOOK_MAPY_PROXY_PATH_PREFIX` if your endpoint lives under a different path.

## GPX preset

```tsx
import { GpxTrackLayer, LeafletMap, parseGpx } from 'react-mapy';

const document = parseGpx(gpxXmlString);

export function GpxExample() {
  return (
    <LeafletMap center={{ lat: 50.0755, lng: 14.4378 }} zoom={13} style={{ height: 420 }}>
      <GpxTrackLayer track={document.tracks[0]} colorMode="pace" />
    </LeafletMap>
  );
}
```

## Theme presets

`LeafletMap` supports built-in presentation themes:

- `mapy`
- `dark`
- `neutral`
- `mapbox`

These presets style the Leaflet surface, controls, popups, attribution, and raster tile treatment. They do not switch the underlying tile provider.

## Georeferenced rasters

`GeoreferencedImageOverlay` lets you place a browser-friendly raster above the map using either direct geographic bounds or `image + world file` metadata.

The library currently focuses on the practical `PNG/JPG + pgw/jgw/tfw` workflow. GeoTIFF decoding is intentionally left outside the runtime bundle.

## Heatmaps

`HeatmapLayer` covers both common density overlays:

- historical event locations below a current marker
- historical route density below an active `StreamTrackLayer`

Use `buildTrackHeatmapPoints(...)` to turn archived paths into a continuous heat surface that darkens where many past routes overlap.

## Clustered events

`MarkerClusterLayer` is intended for event feeds or venue schedules where several records can share one map point.

```tsx
import 'leaflet.markercluster/dist/MarkerCluster.css';
import 'leaflet.markercluster/dist/MarkerCluster.Default.css';

import { LeafletMap, MarkerClusterLayer, loadLeafletMarkerCluster } from 'react-mapy';

await loadLeafletMarkerCluster();

const events = [
  { id: 'evt-1', popupText: 'Morning concert setup', position: { lat: 50.0755, lng: 14.4378 } },
  {
    id: 'evt-2',
    popupText: 'Conference registration opens',
    position: { lat: 50.0755, lng: 14.4378 },
  },
  { id: 'evt-3', popupText: 'Gallery opening', position: { lat: 50.0762, lng: 14.4404 } },
];

export function ClusterExample() {
  return (
    <LeafletMap center={{ lat: 50.0755, lng: 14.4378 }} zoom={13} style={{ height: 420 }}>
      <MarkerClusterLayer markers={events} />
    </LeafletMap>
  );
}
```

Storybook includes a dedicated example in `Leaflet/MarkerClusterLayer -> ClusteredEvents`.

For feed-oriented map screens, Storybook also includes:

- `Leaflet/MarkerClusterLayer -> Custom cluster icons with visible items callback`

That example shows how to:

- render branded cluster badges through `clusterIcon`
- derive a synchronized result list through `onVisibleItemsChange`

## Responsive marker size

`MarkerLayer` and `MarkerClusterLayer` both support zoom-aware icon sizing through `customIcon.size`.

```tsx
<MarkerLayer
  customIcon={{
    preset: 'ofeed',
    size: (zoom) => (zoom >= 15 ? [40, 60] : [24, 36]),
  }}
  position={{ lat: 50.0755, lng: 14.4378 }}
/>
```

Use this when a branded marker should stay visually quieter on overview zooms and only grow at closer inspection levels.

Storybook includes:

- `Leaflet/MarkerLayer -> Responsive marker size by zoom`

## React popup content

`MarkerLayer` and clustered marker entries can render static React markup into Leaflet popups and tooltips.

```tsx
<MarkerLayer
  popupContent={
    <div>
      <strong>Prague city center</strong>
      <span>Static React popup content</span>
    </div>
  }
  position={{ lat: 50.0755, lng: 14.4378 }}
  tooltipContent={<span>React tooltip content</span>}
/>
```

This is useful when plain strings stop being enough, but you still want to keep the runtime lightweight.

Storybook includes:

- `Leaflet/MarkerLayer -> React popup and tooltip content`

## Fit To Data

`FitToData` is a small controller component that computes merged bounds from mixed inputs and applies one `fitBounds` call.

```tsx
<LeafletMap center={pragueCenter} zoom={13} style={{ height: 420 }}>
  <FitToData
    geoJson={featureCollection}
    paths={[trackPoints]}
    points={events.map((event) => event.position)}
    fitBoundsOptions={{ padding: [28, 28] }}
  />
</LeafletMap>
```

Use this when the host screen combines markers, tracks, and overlays but should make just one explicit viewport decision.

Storybook includes:

- `Leaflet/LeafletMap -> Fit viewport to mixed map data`

## Layer architecture

- `core` exposes provider contracts, geometry helpers, and pace utilities.
- `providers/mapy` isolates Mapy.com-specific tile URL building and capabilities.
- `leaflet` contains React wrappers that manage Leaflet maps and layers.
- `parsers` stays DOM-free so GPX and georeferenced asset parsing remain unit-testable.
- `presets` packages higher-level defaults such as pace-colored GPX rendering.
